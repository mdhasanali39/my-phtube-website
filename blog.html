Introduction

Before es6, JavaScript only had global scope and function scope. And then there was only the var keyword for variable declaration. But after es6 came with es6 two keywords let and const, and let and const are blocked scope but load was function scope. Because if you declare a variable with load, it becomes global and gets hosted, but if you declare a load variable only inside a function, it gets hosted and goes to the top of the function.

Scope

Basically scope means from where we can use the variable we have declared/doing.

Var vs Let vs Const

  • As a result, any variable declared outside of a function block using the var declaration syntax is accessible throughout the entire window.
  • When a variable is declared inside a function, it has a function scope. This indicates that it is accessible and limited to that function.
  • While let and const are block scoped, var declarations are either globally scoped or function scoped.
  • Let variables can be updated but not re-declared, const variables cannot be updated or re-declared, and var variables can both be updated and re-declared inside their scope.
  • Each of them is hoisted to the top of their scope. However, let and const variables are not initialized, but var variables are initialized with undefined.
  • with const, which needs to be initialized during declaration, var and let can be declared without being initialized.
